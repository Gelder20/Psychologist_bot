# Psychologist_bot, бот, который по средству общения с ним, анкетирования и опросов выявляет эмоциональное состояние клиента

# В боте реализовано
- БД на postgreSQL, сохраняющая диалог, результаты анкетирования и результат опроса
- Служба поддержки и помощи по боту
- Общения с ботом
- Оценка эмоционального окраса сообщений при общении, с целью создания экстренных вызовов специалиста
- Опросы и анкетирование

# Psychologist_bot, описание и инструкция
Psychologist_bot бот для общения с клиентом, с целью оценки его эмоционального состояния и его дальнейшого улучшения.

**От лица клиента:**

Когда клиент запускает бота он проходит опрос из различных вопросов с 4 вариантами ответа, после оценивает своё эмоциональное самочуствие от 1 до 10. Далее клиенту предлагают начать диалог и начинается процесс общения с ботом. Во время общения можно закончить диалог или вызвать специалиста службы поддержки. Если закончить диалог, будет повторно вызвана анкета и после бот выведет результат его работы (стало ли хуже, лучше или ничего не изменилось), если обратиться в поддержку начнётся процесс общения специалиста с клиентом через бота. Также, если бот выявит плохой эмоциональный окрас сообщения во время общения клиента с ботом, то специалист будет вызван автоматически.



# Описание файлов

Корневая папка:

- main - главный файл

- db - файл для управления БД

- config - файл с важными константами 

- psylogbot - json файл для библиотеки Dialogflow

classes:

- person_manager - управляет различными видами пользователей

- mode_switch - позволяет переключать и управлять режимами

classes.persons:

- person - стандартный класс, от него наследуются все остальные персоны

- user - обычный пользователь

- assistant - ассистент 

classes.modes:

- basic - стандартный класс, от него наследуются все остальные режимы

- broadcast - режим отправки всех сообщений другому пользователю (нужен для связи с тех поддержкой)

- dialogue - режим разговора с ботом

- interview - психологический тест/вопросы

- last_question - вопрос "как вы себя чувствуете"

- waiting_help - ожидание помощи userом

- waiting_request -  если ассистент еще не соединен с кем-либо
