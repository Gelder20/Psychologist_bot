# Psychologist_bot, краткое описание возможностей

- БД на postgreSQL, сохраняющая диалог, клиентов и эмоциональную оценку
- Служба поддержки по боту
- dostoevsky. Оценка эмоционального окраса сообщений при общении, для выявления экстренных вызовов специалиста
- Dialogflow. Общения с ботом
- Опросы и анкетирование

# Psychologist_bot, общее краткое описание
Бот для общения с клиентом, с целью оценки его эмоционального состояния.

В боте реализовано 2 главных режима работы - это специалист и клиент. Чтобы бот различал от друг-друга пользователей, telegram-id специалистов хранятся в конфиге, в переменной (ASSISTANTS).

![изображение](https://user-images.githubusercontent.com/104253966/177141477-c36b920e-8ea7-4301-a3a5-8385a308ad16.png)

Чтобы начать работу с ботом достаточно прописать /start, далее если Вы клиент, запустится опрос для оценки эмоционального состояния и дальнейшие события, а если Вы специалист, бот будет ждать пока кому-то понадобится помощь специалиста и вышлет вам уведомление, если она кому-то понадобится, также реализована возможность иметь одновременно большое кол-во специалистов.

![изображение](https://user-images.githubusercontent.com/104253966/177143110-b3bd088b-5641-4b4a-9207-e57a0f8eb40b.png)

Если специалист откликается, начинается процесс его общения с клиентом, которому нужна помощь, через бота.

Для клиента процесс взаимодействия с ботом следующий:

- Опрос
- Анкетирование
- Общение
- Повторное анкетирование

Процесс общения реализован при помощи библиотеки Dialogflow

# Описание файлов

Корневая папка:

- main - главный файл

- db - файл для управления БД

- config - файл с важными константами 

- psylogbot - json файл для библиотеки Dialogflow

classes:

- person_manager - управляет различными видами пользователей

- mode_switch - позволяет переключать и управлять режимами

- classes.persons:

- person - стандартный класс, от него наследуются все остальные персоны

- user - обычный пользователь

- assistant - ассистент 

- classes.modes:

- basic - стандартный класс, от него наследуются все остальные режимы

- broadcast - режим отправки всех сообщений другому пользователю (нужен для связи с тех поддержкой)

- dialogue - режим разговора с ботом

- interview - психологический тест/вопросы

- last_question - вопрос "как вы себя чувствуете"

- waiting_help - ожидание помощи userом

- waiting_request -  если ассистент еще не соединен с кем-либо
